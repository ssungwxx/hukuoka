# Fukuoka Route & Transit Skill

이 skill은 후쿠오카 여행 프로젝트의 교통편과 동선을 관리한다. 여행 일정 내 장소 간 이동 경로를 분석하고, 최적 교통수단과 비용을 제공한다.

## Data Source

모든 교통 정보는 `/Users/felix/personal/projects/hukuoka/src/app/page.tsx`에 있다.

- `transportRoutes` 배열: 검증된 교통 경로 데이터
- `day1Items` ~ `day5Items` 배열: 일별 일정과 좌표 데이터
- `TransportRoute` 타입: `{ from, to, method, time, cost }`

작업 시작 시 반드시 `page.tsx`를 읽어 최신 데이터를 확인한다.

## How to Analyze Routes

### 1. "X에서 Y까지 어떻게 가?" 질문 처리

최적 경로를 단계별로 제시한다:

**Example:**
> "오호리공원에서 나나쿠마역까지 어떻게 가?"

Answer:
1. 오호리공원역에서 지하철 공항선 탑승 → 텐진역 (4분, ¥210)
2. 텐진역에서 도보 이동 → 텐진미나미역 (약 3분)
3. 텐진미나미역에서 나나쿠마선 탑승 → 나나쿠마역 (12분, ¥300)
- **총 소요**: 약 19분 + 대기시간
- **총 비용**: ¥510

항상 포함할 정보:
- 환승 상세 (노선명, 방향)
- 비용과 소요시간
- IC카드 호환: Hayakaken, nimoca, SUGOCA, Mobile Suica 모두 사용 가능 (후쿠오카 지하철, 니시테츠, JR 큐슈)

### 2. 하루 일정 동선 분석

하루 일정을 받으면 다음을 수행한다:

1. **page.tsx에서 해당 일자의 일정 데이터를 읽는다** (dayNItems 배열)
2. **순서대로 구간별 이동시간/비용을 계산한다**
3. **비효율 패턴을 감지한다**:
   - 왕복(back-and-forth): 같은 지역을 두 번 지나가는 경우
   - 비효율 환승: 직행이 가능한데 환승하는 경우
   - 도보 가능 구간: 1km 이내이면 도보를 권장
4. **개선안이 있으면 재배치를 제안한다** (시간 절약량 명시)
5. **총 교통비와 총 이동시간을 합산한다**

### 3. 교통비 총액 산출

일별 또는 전체 여행의 교통비를 합산할 때:
- transportRoutes의 cost 값을 기반으로 계산
- "~" 가 붙은 요금은 최대값으로 계산
- "무료" 구간은 0으로 처리
- IC카드 이용 시 약간의 할인이 있을 수 있음을 언급

### 4. 동선 최적화 제안

일정 재배치를 제안할 때의 원칙:
- 같은 지역의 장소를 묶는다 (예: 텐진 근처를 연속 방문)
- 식사 시간대(점심 11:30-13:00, 저녁 18:00-20:00)를 유지한다
- 예약이 있는 일정은 고정한다
- 이동 방향을 한 방향으로 유지한다 (예: 동→서 또는 서→동)

## Output Format

경로 안내 시 다음 형식을 따른다:

### 단일 경로
```
🚇 [출발지] → [도착지]
   노선: [노선명]
   소요: [시간] | 비용: [금액]
```

### 하루 동선 요약 (테이블)
| 순서 | 구간 | 교통수단 | 시간 | 비용 |
|------|------|----------|------|------|
| 1 | A → B | 지하철 | 5분 | ¥260 |
| 2 | B → C | 도보 | 10분 | 무료 |
| **합계** | | | **15분** | **¥260** |

### 경로 비교 (복수 옵션)
| 옵션 | 경로 | 시간 | 비용 | 비고 |
|------|------|------|------|------|
| A | 지하철 직행 | 15분 | ¥260 | 추천 |
| B | 버스 → 도보 | 25분 | ¥100 | 저렴 |

## Updating Routes

새로운 교통 경로를 추가할 때:
1. `page.tsx`의 `transportRoutes` 배열에 `{ from, to, method, time, cost }` 형식으로 추가
2. 이 SKILL.md의 해당 섹션 테이블에도 동기화
3. WebSearch로 최신 요금/시간을 검증할 수 있다면 검증 후 추가

## Real-Time Transit Lookup

검증된 경로 데이터에 없는 구간이 요청되면:
1. 먼저 기존 경로를 조합해 환승 경로를 만들 수 있는지 확인
2. 불가능하면 WebSearch로 Google Maps 또는 Navitime 경로를 조회
3. 조회 결과를 사용자에게 제공하되, 검증되지 않은 데이터임을 명시
4. 사용자가 확인하면 transportRoutes에 추가
